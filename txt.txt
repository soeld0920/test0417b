git
버전관리 스시템
버전관리 : 여러 파일을 하나의 전으로 묶어 관리

클라이언트 - 서버 모델
하나의 중앙 서버 여러 클라이언트들이 각자 필요한 데이터만 가져와 작업을 하고 다시 중앙 서버에 합치는 방식
CVS, SVN

분산 모델
하나의 중앙 서버가 존재하지만 여러 클라이언트들은 각자의 컴퓨터 저장소에 전체 사본을 가지고 작업하는 방식
git

GIT의 장점
  동시에 작업하는 사람들과 소스코드를 주고 받을 수 없음
  같은 파일을 여러명이 동시 병렬 개방 가능
  채게적 관리 및 이전 시점으로 되돌리기 가능
  인터넷 연결 없이도 개발 진행이 되며 중앙 서버의 데이터가 유실되어도 다시 복구함



구글에서 git 설치
download 클릭
OS에 따른 프로그램 자운로드
확인(cmd에 git 명령어)
기본설정으로 설치
  터미널 기본 명령어
  ctrl+L 화면 클리어
  pwd 디렉터리 경로 표시
  ls 현 디렉토리에 존재하는 파일 표시
  cd 다른 디렉토리로 이동
  cd,, 상위 디렉토리로
  cd 디렉토리명 해당 디렉토리로 이동
git 로컬 저장소 새엇ㅇ
git에 버전 관리 파일 생성
  txt.txt, index.html
버전관리 파일 선택
  파일 하나만 선택 : git add 파일명.확장자 
  파일 여러개 선택 : git add .
버전관리 파일 커밋
  git commit -m "index.html 버전관리"
로그 확인
  git log
브런치명 설정
  git pull origin 브랜치명(main)
로컬 저장소에 github 저장소 주소 설정
  git remote add orgin git%ithub.com:~~~

  연결 상태
  git remote -v
  연결 해제
  git remote remove origin
만든 커밋 github 무시
  git push -u origin master
  git push origin master
이전 연결 github를 사용
  git add . (재선택)
  git commit -m "메세지" (커밋 대기)
  git pull origin main (브런치명 설정)
  git push -u origin main (push)
